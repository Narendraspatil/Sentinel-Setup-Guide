🛡️ Sentinel – Complete Setup Guide

Zero to a fully running system: cloud server + web dashboard + Windows .exe agent

🪟 Windows 10/11 🐍 Python 3.10+ ☁️ Render.com (Free) ⏱ ~45 minutes total
OVERVIEW

What You'll Build

☁️ Cloud Server + Dashboard

A FastAPI backend hosted on Render.com (free). It stores all logs, controls modes, and serves the web dashboard at a public URL you can access from any phone or browser.

🖥️ Windows Agent (.exe)

A single SentinelAgent.exe file that installs as a Windows service on your laptop. Auto-starts on boot, no Python needed to run it.

ℹ️

Order matters: Deploy the server first, get your server URL, then configure and build the agent. You need the server URL before building the .exe.

BEFORE YOU START

What You Need

Accounts to Create (all free)

  • GitHub account – github.com — to store your code (Render deploys from GitHub)
  • Render.com account – render.com — to host your server for free

Software on Your Windows PC

  • Python 3.10+ — only needed to build the .exe (not to run it)
  • Git — to push code to GitHub. Download at git-scm.com
  • A terminal — Windows Command Prompt or PowerShell (as Administrator for service install)

PHASE 1

Step 1 — Push Code to GitHub

⚠️

Render.com deploys directly from a GitHub repository. You must push your code to GitHub first. This is a one-time step.

1A

Create a new GitHub repository

  1. Go to github.com/new
  2. Name it sentinel (or anything you like)
  3. Set it to Private (so your security config isn't public)
  4. Click Create repository
  5. GitHub shows you a page with commands — leave this tab open
1B

Push your sentinel folder to GitHub

Open Command Prompt or PowerShell. Navigate into your sentinel folder:

# Navigate to your extracted sentinel folder
cd C:\Users\YourName\Downloads\sentinel

# Initialize git and push
git init
git add .
git commit -m "Initial commit – Sentinel project"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/sentinel.git
git push -u origin main
ℹ️

Replace YOUR-USERNAME with your actual GitHub username. Git will ask for your GitHub username and password (or a Personal Access Token if you have 2FA enabled).

PHASE 1

Step 2 — Deploy Server on Render.com

2A

Sign up / Log in to Render

  1. Go to render.com
  2. Click Get Started for Free
  3. Sign up with your GitHub account (easiest — it connects automatically)
2B

Create a new Web Service

  1. In your Render dashboard, click + New → Web Service
  2. Click Connect a repository
  3. Find and select your sentinel repo
  4. Click Connect
2C

Fill in the service settings

On the configuration page, fill in these fields exactly:

Field Value to enter
Name sentinel-server
Region Choose closest to you
Branch main
Runtime Python 3
Build Command pip install -r requirements.txt
Start Command uvicorn server.main:app --host 0.0.0.0 --port $PORT
Instance Type Free ← make sure this is selected
⚠️

Do not click "Create Web Service" yet. You need to add environment variables first (next step).

PHASE 1

Step 3 — Set Environment Variables

3A

Scroll down to "Environment Variables" on the Render setup page

Still on the same Render configuration page, scroll down and find the Environment Variables section. Add these three variables by clicking Add Environment Variable for each:

Key Value Notes
SECRET_KEY Generate one (see below) Signs JWT tokens
DEVICE_API_TOKEN Make up a strong password Agent uses this to authenticate — write it down!
TOKEN_EXPIRE_MINUTES 1440 Dashboard session = 24 hours
3B

Generate a SECRET_KEY

Open Python on your PC and run this one-liner to generate a secure random key:

# Open Command Prompt and run:
python -c "import secrets; print(secrets.token_hex(32))"

# Example output (yours will be different):
a3f8c2e1d4b7a9f0e2c5d8a1b4f7e0c3a6d9b2e5f8c1d4a7b0e3f6c9d2a5b8e1

Copy that output. Paste it as the value for SECRET_KEY in Render.

3C

Choose a DEVICE_API_TOKEN

This is like a password your agent uses to talk to the server. Make something strong and write it down — you'll need it again when configuring the agent.

# Generate one the same way:
python -c "import secrets; print(secrets.token_urlsafe(24))"

# Example output:
K9mXqR2vLwZpN7bYcT4sAeD1
✅

After adding all 3 environment variables, scroll back up and click "Create Web Service". Render will start deploying. This takes 2–4 minutes.

3D

Get your server URL

After deployment finishes (you'll see Live in green), your server URL is shown at the top of the page:

# Your URL will look like this:
https://sentinel-server.onrender.com

# Or with a random suffix:
https://sentinel-server-abc1.onrender.com
📋

Copy and save this URL. You'll need it in Phase 2 when configuring the agent.

PHASE 1

Step 4 — Create Your Admin Account

4A

Open your server's API page

Go to your server URL and add /docs at the end:

https://sentinel-server.onrender.com/docs

This opens the automatic API documentation (Swagger UI). You'll see a list of all API endpoints.

4B

Create admin account via /auth/setup

  1. In the /docs page, scroll to find POST /auth/setup
  2. Click it → click "Try it out"
  3. Replace the example body with your details:
{
  "username": "admin",
  "password": "YourStrongPassword123!"
}
  1. Click Execute
  2. You should see: {"detail": "Admin created successfully."}
ℹ️

This endpoint only works once. After the first admin is created, it returns an error. This is a security feature.

4C

Test your dashboard

Visit your server URL (without /docs):

https://sentinel-server.onrender.com

You should see the Sentinel login page. Log in with the username and password you just created.

🎉

Phase 1 complete! Your server and dashboard are live. Now let's set up the Windows agent.

PHASE 2

Step 5 — Install Python on Windows

ℹ️

Python is only needed on your PC to build the .exe file. Once the .exe is built, you can run it on any Windows machine — no Python required there.

5A

Download and install Python

  1. Go to python.org/downloads
  2. Click Download Python 3.12.x (latest 3.x)
  3. Run the installer
  4. IMPORTANT: On the first screen, check ✅ "Add Python to PATH" before clicking Install Now
5B

Verify Python installed correctly

Open a new Command Prompt and run:

python --version
# Should print: Python 3.12.x

pip --version
# Should print: pip 24.x from ...
⚠️

If you get "python is not recognized", close the Command Prompt and open a new one. If still failing, re-run the Python installer and check the "Add to PATH" box.

PHASE 2

Step 6 — Install Python Dependencies

6A

Navigate to your sentinel folder

cd C:\Users\YourName\Downloads\sentinel
# Replace the path with wherever you extracted the project
6B

Install all dependencies

pip install -r requirements.txt
# This will take 2-5 minutes — downloads all packages
⚠️

If you see an error about pywin32, run this extra command:
pip install pywin32 then python Scripts/pywin32_postinstall.py -install

6C

Install PyInstaller (for building the .exe)

pip install pyinstaller
# Verify it installed:
pyinstaller --version
# Should print: 6.x.x
PHASE 2

Step 7 — Configure the Agent

7A

Edit the default server URL in config.py

Open sentinel/agent/config.py in Notepad or any text editor. Find this line and replace it with your Render URL:

# Find this line (around line 18):
"server_url": "https://your-sentinel-server.onrender.com",

# Change it to YOUR actual Render URL:
"server_url": "https://sentinel-server-abc1.onrender.com",

Save the file.

7B

Run the setup wizard

Now run the interactive setup script to set your token and register the device:

cd sentinel\agent
python setup_agent.py

The wizard will ask you 3 questions. Here's what to enter:

Prompt What to enter
Server URL [current]: Press Enter (already set in 7A), or paste your URL again
Device name [current]: Give your laptop a name, e.g. Johns-Laptop
Device API token: Paste the DEVICE_API_TOKEN you set on Render (Step 3B)

If the connection test succeeds, you'll see: ✅ Device registered successfully!

⚠️

If you see ❌ Authentication failed, double-check that the token you entered here exactly matches the DEVICE_API_TOKEN you set in Render's environment variables.

PHASE 2

Step 8 — Build the .exe File

ℹ️

PyInstaller bundles your Python scripts + all dependencies into a single .exe file. This takes 2–5 minutes.

8A

Go back to the root sentinel folder and run PyInstaller

# Navigate to the root sentinel folder (not the agent subfolder)
cd C:\Users\YourName\Downloads\sentinel

# Build using the spec file
pyinstaller sentinel_agent.spec
8B

Find your .exe

After PyInstaller finishes, the executable is at:

sentinel\dist\SentinelAgent.exe

This is the only file you need to install the service. You can copy it anywhere on your PC.

8C

If the build fails

The most common issue is a missing import. Try this alternative build command that works more reliably:

# Alternative: build directly (skip the .spec file)
pyinstaller ^
  --onefile ^
  --name SentinelAgent ^
  --hidden-import win32timezone ^
  --hidden-import watchdog.observers.winapi ^
  --paths agent ^
  agent\service.py
⚠️

If you get errors about cv2 (OpenCV) not found, install it: pip install opencv-python and try again.

PHASE 2

Step 9 — Install as Windows Service

🔑

This step requires Administrator privileges. Right-click Command Prompt → "Run as administrator".

9A

Open Administrator Command Prompt

  1. Press Windows key
  2. Type cmd
  3. Right-click Command Prompt → "Run as administrator"
  4. Click Yes on the UAC prompt
9B

Navigate to the .exe and install

# Navigate to where your .exe is
cd C:\Users\YourName\Downloads\sentinel\dist

# Install the Windows service
SentinelAgent.exe install

# Start the service
SentinelAgent.exe start
9C

Verify the service is running

# Check service status
sc query SentinelAgent

# You should see:
# STATE : 4  RUNNING

You can also open Task Manager → Services tab and search for SentinelAgent.

✅

The service is now installed and will auto-start every time Windows boots. If the service crashes, Windows will automatically restart it (recovery is configured during install).

9D

Service management commands (for reference)

# Stop the service
SentinelAgent.exe stop

# Restart the service
SentinelAgent.exe restart

# Remove the service entirely
SentinelAgent.exe remove

# Check logs (stored here)
type %APPDATA%\..\Local\sentinel\agent\data\logs\sentinel_service.log

PHASE 3

Step 10 — Log Into Your Dashboard

Access from anywhere

Open any browser on any device (phone, tablet, another computer) and go to:

https://sentinel-server.onrender.com

Log in with the admin username and password you created in Step 4.

After a few seconds (up to 15), your device should appear in the Dashboard and show as Online.

⚠️

First load may be slow (30–60 sec). Render's free tier "spins down" after 15 minutes of inactivity. The first request wakes it up. Subsequent loads are instant.

PHASE 3

Step 11 — Enable Secure Mode

Toggle mode from the dashboard

  1. On the Dashboard page, find your device in the table
  2. Click "Set Secure" button next to it
  3. Confirm the dialog
  4. The mode badge changes to SECURE
  5. Within 10 seconds, the agent on your laptop receives the change and activates protection

What happens in Secure Mode

  • Blocked apps are killed immediately if they launch
  • Any file write/create in protected folders is detected and reverted
  • A webcam photo is captured and uploaded on every security event
  • All events are logged and visible in the Logs tab
PHASE 3

Step 12 — Configure Blocked Apps & Protected Folders

Go to Settings tab

  1. Click Settings in the sidebar (or bottom nav on mobile)
  2. Select your device from the dropdown
  3. Two panels appear: Blocked Applications and Protected Folders

Blocked Applications

Enter executable names (one per line). Example:

chrome.exe
excel.exe
outlook.exe
teams.exe
discord.exe

In Secure Mode these processes will be killed instantly.

Protected Folders

Enter full Windows paths (one per line). Example:

C:\Users\John\Documents\Private
C:\Users\John\Desktop\Secrets
D:\Work\Confidential

Any file changes here trigger an alert + photo.

Save settings

Click Save on each panel. Changes sync to the agent within 10 seconds (the next poll cycle).

🎉

You're all set! Sentinel is fully deployed and running. Your laptop is now remotely protected.

TROUBLESHOOTING

Common Issues & Fixes

🔴 Device shows as Offline in dashboard

  • Check the service is running: sc query SentinelAgent
  • Check the log file for errors: type C:\Users\YourName\sentinel\agent\data\logs\sentinel_service.log
  • Verify your server URL in agent/config.py and in agent/data/agent_config.json
  • Try pinging the server: open browser → go to your Render URL → you should see the login page

🔴 Render deploy failed (red "Failed" status)

  • Click on the deploy → scroll down to see the build log
  • Most common cause: wrong Start Command. Make sure it's exactly: uvicorn server.main:app --host 0.0.0.0 --port $PORT
  • Check that requirements.txt is in the root of the repo (not inside a subfolder)

🔴 "python is not recognized" when installing

  • Re-run the Python installer → check "Add Python to PATH" → restart your PC
  • Or try using py instead of python in all commands

🔴 PyInstaller build fails with import errors

# Run this to get more detailed error info:
pyinstaller sentinel_agent.spec --log-level DEBUG

# Fix missing imports by adding --hidden-import:
pip install <missing-package-name>

🔴 Service install says "Access Denied"

You must run Command Prompt as Administrator:

  • Search "cmd" in Start menu
  • Right-click → Run as administrator
  • Try the install command again

🔴 Dashboard loads but no devices appear

  • Wait 15 seconds then refresh — the agent polls every 10 seconds
  • Make sure you ran setup_agent.py before building the .exe
  • Check Render logs: Render dashboard → your service → Logs tab — look for registration requests

🔴 Camera not capturing photos

  • Ensure opencv-python is installed: pip install opencv-python
  • Test your camera: python -c "import cv2; cap = cv2.VideoCapture(0); print('OK' if cap.isOpened() else 'No camera')"
  • Close any other apps using the camera (Zoom, Teams, etc.)
  • Rebuild the .exe after installing opencv
