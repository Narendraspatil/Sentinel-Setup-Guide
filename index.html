🛡️ Sentinel – Complete Setup Guide

Zero to a fully running system: Railway server + PostgreSQL + web dashboard + Windows agent

🪟 Windows 10/11 🐍 Python 3.10+ 🚂 Railway.app 🐘 PostgreSQL ⏱ ~45 minutes
OVERVIEW

What You'll Build

☁️ Cloud Server + Dashboard

A FastAPI backend on Railway with a PostgreSQL database. Stores all logs and photos, controls device modes, and serves the web dashboard at a permanent public URL.

🖥️ Windows Agent

Python scripts that run as a Windows Service on your laptop. Auto-starts on boot. Polls the server every 10 seconds for mode changes, buffers logs offline, and captures webcam photos on security events.

ℹ️

Order matters: Deploy the server first and get your server URL + REGISTRATION_TOKEN. You need both before configuring the agent.

BEFORE YOU START

SECURITY

How Device Authentication Works

Per-device secrets (not a shared token)

The old system used one shared DEVICE_API_TOKEN for all agents — a single leaked token would compromise every device. The new system gives each device its own unique secret:

#WhoWhat happens
1You (Railway)Set REGISTRATION_TOKEN env var — a one-time secret for first registration only
2Agent (first run)Calls POST /api/devices/register with the REGISTRATION_TOKEN
3ServerGenerates a unique device_secret (32 random bytes), stores only the bcrypt hash
4Server → AgentReturns device_secret in plaintext once — agent saves it locally
5Agent (all future calls)Uses Authorization: Bearer <device_id>:<device_secret>
6ServerLooks up device by ID, verifies secret against bcrypt hash — never stores plaintext
ℹ️

If a device is compromised, you delete it from the dashboard and re-register. Other devices are completely unaffected.

PHASE 1

Step 1 — Push Code to GitHub

⚠️

Railway deploys directly from a GitHub repository. Push your code first — this is a one-time step.

1A

Create a new GitHub repository

  1. Go to github.com/new
  2. Name it sentinel
  3. Set it to Private — your security config shouldn't be public
  4. Click Create repository
1B

Push your sentinel folder to GitHub

# Navigate to your sentinel folder
cd C:\Users\YourName\Downloads\sentinel

# Initialize and push
git init
git add .
git commit -m "Initial commit – Sentinel project"
git branch -M main
git remote add origin https://github.com/YOUR-USERNAME/sentinel.git
git push -u origin main
ℹ️

Replace YOUR-USERNAME with your GitHub username. Git may ask for your credentials or a Personal Access Token if 2FA is enabled.

PHASE 1

Step 2 — Create Railway Project

2A

Sign up / Log in to Railway

  1. Go to railway.app
  2. Click Login → sign in with GitHub (connects automatically)
  3. On the Dashboard, click + New Project
2B

Deploy from GitHub repo

  1. Choose Deploy from GitHub repo
  2. Select your sentinel repository
  3. Railway will detect the project and create a service — do not click Deploy yet
⚠️

The first deploy will fail until you add the PostgreSQL database and environment variables. That's expected — continue to the next steps.

PHASE 1

Step 3 — Add PostgreSQL Database

ℹ️

Sentinel v2 uses PostgreSQL only — no SQLite. Railway's PostgreSQL plugin sets up the database and injects DATABASE_URL automatically.

3A

Add the PostgreSQL plugin

  1. In your Railway project, click + New
  2. Select Database → Add PostgreSQL
  3. Railway provisions a PostgreSQL instance in ~30 seconds
  4. Click on the new PostgreSQL service → go to the Variables tab
  5. You'll see DATABASE_URL listed — Railway automatically shares this with your web service
✅

You don't need to copy or paste the DATABASE_URL. Railway injects it automatically into your web service's environment.

PHASE 1

Step 4 — Set Environment Variables

4A

Open your web service's Variables tab

In Railway, click on your web service (not the PostgreSQL service) → click the Variables tab → click + New Variable to add each one.

VariableValueNotes
SECRET_KEY Generate — see 4B Signs JWT tokens for dashboard login
REGISTRATION_TOKEN Generate — see 4C One-time token agents use to register
ALLOW_SETUP true Enables /auth/setup to create admin. Set to false after Step 8.
TOKEN_EXPIRE_MINUTES 1440 Dashboard session length (24h). Optional.
OFFLINE_THRESHOLD_SECONDS 60 Seconds before device marked offline. Optional.
ℹ️

DATABASE_URL is not listed here because Railway injects it automatically from the PostgreSQL plugin you added in Step 3.

4B

Generate SECRET_KEY

Open Command Prompt on your PC and run:

python -c "import secrets; print(secrets.token_hex(32))"

# Example output (yours will be different):
a3f8c2e1d4b7a9f0e2c5d8a1b4f7e0c3a6d9b2e5f8c1d4a7b0e3f6c9d2a5b8e1

Copy the full output and paste it as the value for SECRET_KEY in Railway.

4C

Generate REGISTRATION_TOKEN

This is the one-time token your agents use when registering for the first time. Generate one and write it down — you'll need it in Phase 2.

python -c "import secrets; print(secrets.token_hex(24))"

# Example output:
7f3a9c1e5b8d2f4a6e0c3b7d9a1f4c2e8b5d0a3f
📋

Save the REGISTRATION_TOKEN somewhere safe (password manager, notes app). You'll paste it into the agent setup wizard in Phase 2.

Unlike the old DEVICE_API_TOKEN, this token is only used once per device registration — not for every API call.

PHASE 1

Step 5 — Configure Start Command & Deploy

5A

Set the start command

In Railway → your web service → Settings tab → scroll to Deploy → set the Start Command:

cd server && uvicorn main:app --host 0.0.0.0 --port $PORT
ℹ️

This is also already set in railway.toml and Procfile in the repo root, so Railway may detect it automatically.

5B

Set the build command

In the same Settings → Build section, set:

pip install -r server/requirements.txt

This installs only the server dependencies — agent-specific packages like pywin32 and opencv are excluded.

5C

Trigger a deploy

  1. Click Deploy (or push any commit to GitHub — Railway auto-deploys)
  2. Click on the deploy to watch the build log
  3. A successful deploy ends with: Application started
  4. Railway shows a green checkmark and gives you a public URL
✅

Your URL looks like: https://sentinel-production-xxxx.up.railway.app — copy and save it.

⚠️

If the deploy fails with "Required environment variable 'X' is not set" — go back and add the missing variable in the Variables tab, then redeploy.

PHASE 1

Step 6 — Create Admin Account

⚠️

The /auth/setup endpoint only works when ALLOW_SETUP=true is set in Railway AND no admin exists yet. You set this in Step 4 — it's active now.

6A

Create the admin account via curl or the API docs

Option A — via terminal (curl):

curl -X POST https://your-server.up.railway.app/auth/setup \
  -H "Content-Type: application/json" \
  -d '{"username": "admin", "password": "YourStrongPassword123!"}'

# Success response:
{"detail": "Admin created successfully. Set ALLOW_SETUP=false now."}

Option B — via Swagger UI:

  1. Go to https://your-server.up.railway.app/docs
  2. Find POST /auth/setup → click Try it out
  3. Enter your username and password, click Execute
6B

Immediately disable setup — IMPORTANT

After the admin is created, go back to Railway → your web service → Variables and change:

ALLOW_SETUP = false

Railway will redeploy automatically. The /auth/setup endpoint will now return 403 Forbidden for all future requests — preventing anyone from overwriting your admin account.

🔒

Do not skip this. Leaving ALLOW_SETUP=true is a security hole — anyone who finds your URL could replace your admin account.

6C

Test the dashboard

Visit your server URL (no path, just the root):

https://your-server.up.railway.app

You should see the Sentinel login page. Log in with the credentials you just created.

Phase 1 complete! Your server, database, and dashboard are live on Railway.

PHASE 2

Step 9 — Run the Setup Wizard

ℹ️

This wizard registers your device with the server. It uses the REGISTRATION_TOKEN you generated in Step 4C to prove it's an authorised device — then receives a unique device_secret that's stored locally.

You only run this once per machine. After this, the agent uses the stored secret automatically.

9A

Run the setup wizard

# Make sure you're in the agent folder
cd C:\Users\YourName\Downloads\sentinel\agent

python setup_agent.py
9B

Answer the prompts

The wizard asks three questions:

PromptWhat to enter
Server URL [current]: Paste your Railway URL, e.g. https://sentinel-xxxx.up.railway.app
Device name [current]: A friendly name for this laptop, e.g. Johns-Laptop
REGISTRATION_TOKEN: Paste the token you generated in Step 4C and saved
9C

Successful registration output

If everything is correct you will see:

✅ Device registered successfully!
   Device ID:     xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   Device Secret: ********************  (stored in agent_config.json)

The wizard stores the device_secret in agent/data/agent_config.json. The service reads it automatically from there — you never need to copy it manually.

🔒

agent/data/agent_config.json contains your device credentials. Do not share it, commit it to git, or copy it to other machines. The .gitignore already excludes the agent/data/ folder.

⚠️

If you see ❌ Registration failed: Invalid REGISTRATION_TOKEN — the token you entered doesn't match the one set in Railway. Double-check for extra spaces or typos.

If you see a connection error — verify the server URL is correct and the Railway deploy is live.

PHASE 2

Step 10 — Install as Windows Service

🔑

This step requires Administrator privileges. Right-click Command Prompt → "Run as administrator".

10A

Open an Administrator Command Prompt

  1. Press the Windows key
  2. Type cmd
  3. Right-click Command Prompt → "Run as administrator"
  4. Click Yes on the UAC prompt
10B

Install and start the service

# Navigate to the agent folder
cd C:\Users\YourName\Downloads\sentinel\agent

# Install the Windows service
python service.py install

# Start it
python service.py start
10C

Verify the service is running

sc query SentinelAgent

# You should see:
# STATE              : 4  RUNNING

You can also open Task Manager → Services tab and search for SentinelAgent.

✅

The service auto-starts every time Windows boots. Windows SCM is also configured to restart it automatically if it crashes.

10D

Service management commands (reference)

# Stop the service
python service.py stop

# Restart the service
python service.py restart

# Remove the service entirely
python service.py remove

# View logs
type agent\data\logs\sentinel_service.log

PHASE 3

Step 13 — Enable Secure Mode

Toggle mode from the dashboard

  1. On the Dashboard, find your device in the table
  2. Click "Set Secure" next to it
  3. The mode badge changes to SECURE
  4. Within 10 seconds the agent receives the change and activates protection

What happens in Secure Mode

  • Blocked apps are killed immediately if they launch
  • Any file write/create in protected folders is detected and reverted
  • A webcam photo is captured and uploaded on every security event
  • All events appear in the Logs tab in real time
PHASE 3

Step 14 — Configure Blocked Apps & Protected Folders

Go to the Settings tab

  1. Click Settings in the dashboard sidebar
  2. Select your device from the dropdown
  3. Two panels: Blocked Applications and Protected Folders

Blocked Applications

Enter executable names (one per line):

chrome.exe
excel.exe
outlook.exe
teams.exe
discord.exe

These processes are killed instantly in Secure Mode.

Protected Folders

Enter full Windows paths (one per line):

C:\Users\John\Documents\Private
C:\Users\John\Desktop\Secrets
D:\Work\Confidential

Any file change here triggers an alert + webcam photo.

Save and sync

Click Save on each panel. Changes sync to the agent within 10 seconds on the next poll cycle.

TROUBLESHOOTING

Common Issues & Fixes

🔴 Railway deploy fails with "Required environment variable not set"

  • Go to Railway → your web service → Variables tab
  • Check that SECRET_KEY and REGISTRATION_TOKEN are both added
  • DATABASE_URL should be auto-injected from the PostgreSQL plugin — if it's missing, open the PostgreSQL service and verify it's in the same project
  • After adding missing variables, click Deploy to redeploy

🔴 Device shows Offline in the dashboard

  • Check the service is running: sc query SentinelAgent
  • If it's stopped, check the log: type agent\data\logs\sentinel_service.log
  • A common cause is the agent not being registered — re-run python setup_agent.py
  • Verify the server URL in agent/data/agent_config.json matches your Railway URL exactly

🔴 Agent registration returns 401 Unauthorized

  • The REGISTRATION_TOKEN you entered doesn't match the one set in Railway
  • Open Railway → Variables → copy the exact value of REGISTRATION_TOKEN
  • Re-run python setup_agent.py and paste it again carefully (no extra spaces)

🔴 /auth/setup returns 403 Forbidden

  • This means ALLOW_SETUP is not set to true in Railway — which is the intended behavior after initial setup
  • To create a new admin: temporarily set ALLOW_SETUP=true in Railway → redeploy → create admin → set back to false

🔴 Service install says "Access Denied"

  • You must run Command Prompt as Administrator
  • Search "cmd" in Start menu → right-click → Run as administrator

🔴 Service starts then immediately stops

  • The most likely cause: setup_agent.py was not run before starting the service
  • Check the log: type agent\data\logs\sentinel_service.log — it will say "Agent is not registered"
  • Run python setup_agent.py first, then python service.py start

🔴 Dashboard loads but no devices appear

  • Wait 15 seconds and refresh — the agent polls every 10 seconds
  • Check Railway logs: click your web service → Deployments → View Logs
  • Look for a line like POST /api/device/{id}/config — if absent, the agent isn't reaching the server

🔴 Re-registering a device (rotating credentials)

If you need to reset a device's credentials (e.g. agent_config.json was lost or compromised):

# Simply re-run the setup wizard — it generates a new device_secret
python setup_agent.py

# Then restart the service
python service.py restart

The old secret is overwritten on the server. Other devices are completely unaffected.

🔴 Camera not capturing photos

  • Verify opencv-python is installed: pip install opencv-python
  • Test your camera: python -c "import cv2; cap = cv2.VideoCapture(0); print('OK' if cap.isOpened() else 'No camera')"
  • Close any other apps using the camera (Teams, Zoom, etc.)
